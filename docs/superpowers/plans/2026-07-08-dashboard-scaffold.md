### Task 4: Dockerfile for piper hosting

**Files:**
- Create: `Dockerfile`, `.dockerignore`

**Interfaces:**
- Consumes: `bun run build` producing `.output/server/index.mjs` (Task 1; if Task 1 Step 3 found a different entry path, use that in `CMD` — mirror the scaffold's `start` script).
- Produces: image serving HTTP on **8080** — the contract piper's deploy flow expects.

- [ ] **Step 1: Write `.dockerignore`**

```
node_modules
.output
.git
docs
*.md
```

- [ ] **Step 2: Write `Dockerfile`**

```dockerfile
FROM oven/bun:1 AS build
WORKDIR /app
COPY package.json bun.lock ./
RUN bun install --frozen-lockfile
COPY . .
RUN bun run build

FROM oven/bun:1-slim
WORKDIR /app
COPY --from=build /app/.output .output
ENV PORT=8080
EXPOSE 8080
CMD ["bun", ".output/server/index.mjs"]
```

(If the lockfile is `bun.lockb` rather than `bun.lock`, copy that name instead.)

- [ ] **Step 3: Build and run the image, verify 8080**

```bash
docker build -t piper-dashboard-dev . && \
docker run -d --rm -p 8080:8080 --name piper-dashboard-dev piper-dashboard-dev && \
sleep 2 && curl -s http://localhost:8080 | grep -i "piper dashboard"; docker stop piper-dashboard-dev
```

Expected: `docker build` succeeds; curl output contains the Piper Dashboard heading markup. If curl returns nothing, check `docker logs piper-dashboard-dev` — the usual cause is a wrong server entry path in `CMD`.

