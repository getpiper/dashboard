# Phase 0 — GitHub Login + Session Shim Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A browser logs into the dashboard via the relay's GitHub web flow, the credential lives in httpOnly cookies, and the logged-in home lists the account's boxes with connected badges.

**Architecture:** Per the approved spec (`docs/superpowers/specs/2026-07-09-phase0-auth-design.md`): three thin server routes (`/api/auth/login` 302s to the relay, `/api/auth/session` sets cookies from the fragment handoff, `/api/auth/logout` clears them), one `getBoxes` server function proxying `GET {relay}/agents`, and three pages (`/login`, `/auth/callback`, `/`). All testable logic lives in `src/server/` and `src/components/` as plain functions/components; route files are thin wrappers.

**Tech Stack:** TanStack Start 1.168 (server routes via `server.handlers` on `createFileRoute`; server fns via `createServerFn`; cookie helpers from `@tanstack/react-start/server`), Bun test + happy-dom + Testing Library, Tailwind, Biome.

## Global Constraints

- Bun only — never npm/yarn/node. Run `bun run format` before every commit (Biome enforces tabs + double quotes).
- Tests NEVER live in `src/routes/` (the file router scans it). Colocate tests with `src/server/` and `src/components/` code.
- No hardcoded `getpiper.co` anywhere in the app — relay URL comes only from the `PIPER_RELAY_URL` env var; missing → throw.
- Cookies: `piper_session` (credential) and `piper_username`, both `HttpOnly; Secure; SameSite=Lax; Path=/; Max-Age=2592000`.
- After adding/removing files under `src/routes/`, run `bun run generate-routes` to refresh `src/routeTree.gen.ts`.
- Every commit message is conventional (`feat:`/`test:`/`chore:`) and ends with:
  `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`
- Branch: `ozykhan/phase0-auth` (already exists, spec committed on it).
- `bun run verify` must pass before the final PR (Biome → tsc → tests → build).

**Spec refinement note:** the spec sketches `getBoxes` returning `{username, boxes}`; the header instead gets `username` from a separate `getSession` server fn (root loader), so `getBoxes` returns just `Box[]`. Same data, no duplication.

```ts
export type Box = { agent: string; connected: boolean };

export class RelayAuthError extends Error {}

export function relayUrl(): string {
	const url = process.env.PIPER_RELAY_URL;
	if (!url) {
		throw new Error(
			"PIPER_RELAY_URL is not set — point it at the relay control API (the same URL `piper login --relay` takes)",
		);
	}
	return url.replace(/\/$/, "");
}

export async function fetchBoxes(credential: string): Promise<Box[]> {
	const res = await fetch(`${relayUrl()}/agents`, {
		headers: { Authorization: `Bearer ${credential}` },
	});
	if (res.status === 401) {
		throw new RelayAuthError("relay rejected the session credential");
	}
	if (!res.ok) {
		throw new Error(`relay /agents returned ${res.status}`);
	}
	return (await res.json()) as Box[];
}
```

```ts
const COOKIE_ATTRS = "HttpOnly; Secure; SameSite=Lax; Path=/";
const SESSION_MAX_AGE = 60 * 60 * 24 * 30; // 30 days

export function handleLogin(request: Request, relayBase: string): Response {
	const origin = new URL(request.url).origin;
	const params = new URLSearchParams({
		redirect_uri: `${origin}/auth/callback`,
	});
	return Response.redirect(`${relayBase}/v1/login/web?${params}`, 302);
}

export async function handleSession(request: Request): Promise<Response> {
	let body: unknown;
	try {
		body = await request.json();
	} catch {
		return new Response("invalid JSON body", { status: 400 });
	}
	const { credential, username } = (body ?? {}) as {
		credential?: unknown;
		username?: unknown;
	};
	if (typeof credential !== "string" || credential === "") {
		return new Response("missing credential", { status: 400 });
	}
	const name = typeof username === "string" ? username : "";
	const headers = new Headers();
	headers.append(
		"Set-Cookie",
		`piper_session=${encodeURIComponent(credential)}; ${COOKIE_ATTRS}; Max-Age=${SESSION_MAX_AGE}`,
	);
	headers.append(
		"Set-Cookie",
		`piper_username=${encodeURIComponent(name)}; ${COOKIE_ATTRS}; Max-Age=${SESSION_MAX_AGE}`,
	);
	return new Response(null, { status: 204, headers });
}

export function handleLogout(): Response {
	const headers = new Headers();
	headers.append("Set-Cookie", `piper_session=; ${COOKIE_ATTRS}; Max-Age=0`);
	headers.append("Set-Cookie", `piper_username=; ${COOKIE_ATTRS}; Max-Age=0`);
	return new Response(null, { status: 204, headers });
}
```

```tsx
import { useEffect, useRef, useState } from "react";

// Transient fragment handoff (see the phase-0 design doc): read the
// credential from the URL fragment exactly once, exchange it for httpOnly
// cookies via POST /api/auth/session, and scrub the fragment from history.
export function AuthCallback({ onDone }: { onDone: () => void }) {
	const ran = useRef(false);
	const [failed, setFailed] = useState(false);

	useEffect(() => {
		if (ran.current) return;
		ran.current = true;
		const params = new URLSearchParams(window.location.hash.slice(1));
		const credential = params.get("credential");
		const username = params.get("username") ?? "";
		if (!credential) {
			setFailed(true);
			return;
		}
		window.history.replaceState(null, "", window.location.pathname);
		fetch("/api/auth/session", {
			method: "POST",
			headers: { "Content-Type": "application/json" },
			body: JSON.stringify({ credential, username }),
		})
			.then((res) => {
				if (res.ok) {
					onDone();
				} else {
					setFailed(true);
				}
			})
			.catch(() => setFailed(true));
	}, [onDone]);

	if (failed) {
		return (
			<main className="flex min-h-screen flex-col items-center justify-center gap-3">
				<p>Sign-in failed.</p>
				<a href="/login" className="underline">
					Back to login
				</a>
			</main>
		);
	}
	return (
		<main className="flex min-h-screen items-center justify-center">
			<p className="text-muted-foreground">Signing you in…</p>
		</main>
	);
}
```

- [ ] **Step 5: Write the server functions**

Create `src/server/fns.ts`:

```ts
import { redirect } from "@tanstack/react-router";
import { createServerFn } from "@tanstack/react-start";
import { deleteCookie, getCookie } from "@tanstack/react-start/server";
import { fetchBoxes, RelayAuthError } from "./relay";

export const getSession = createServerFn().handler(async () => {
	const credential = getCookie("piper_session");
	if (!credential) return null;
	return { username: getCookie("piper_username") ?? "" };
});

export const getBoxes = createServerFn().handler(async () => {
	const credential = getCookie("piper_session");
	if (!credential) throw redirect({ to: "/login" });
	try {
		return await fetchBoxes(credential);
	} catch (err) {
		if (err instanceof RelayAuthError) {
			// Revoked/garbage credential: drop the dead session and re-login.
			deleteCookie("piper_session", { path: "/" });
			deleteCookie("piper_username", { path: "/" });
			throw redirect({ to: "/login" });
		}
		throw err;
	}
});
```

```ts
// stub-relay.ts — bun run stub-relay.ts
Bun.serve({
	port: 8099,
	fetch(req) {
		const url = new URL(req.url);
		if (url.pathname === "/v1/login/web") {
			const ru = url.searchParams.get("redirect_uri");
			return Response.redirect(`${ru}#credential=stub-cred&username=zoe`, 302);

			return Response.json([
				{ agent: "abc123-zoe.public.example", connected: true },

			]);

		return new Response("not found", { status: 404 });
	},
});
```
